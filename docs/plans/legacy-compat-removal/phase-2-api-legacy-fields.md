# Phase 2 — Remove Legacy API Fields `params_schema` / `defaults`

**Sprint:** `docs/plans/legacy-compat-removal-sprint.md`
**Phase goal:** Remove the vestigial flat `params_schema` and `defaults`
fields from the `/node-types/{node_type}/schema` endpoint. The frontend
already builds forms from `methods[].params` and never reads these fields.

## Authority

ADR 0003 — params schemas may be reshaped freely; no compat required.

A pre-sprint audit confirmed:
- `sidecar/models.py:747-748` — `NodeTypeSchemaResponse` declares both fields.
- `sidecar/routes/node_types.py:248-276` — the backend synthesizes them from
  `methods[0]` (comment: `# Legacy flat fields from the first available method`).
- The frontend editor (`SchemaDrivenParamsEditor.tsx`) only *normalizes* these
  fields at lines 69-70 and **never reads them afterward**. All form logic
  uses `selectedMethod.params` (lines 121-133, 149-206, 209-267, 424-437).
- `tests/test_explainability_limitations.py:479-518` — 14 real assertions on
  the synthesized shape. **Decision: rewrite them 1:1 against
  `methods[0].params`, not delete them.**
- Frontend mocks (`SchemaDrivenParamsEditor.test.tsx`) include the fields as
  empty stubs but assert nothing on them.

## Steps

### Step 1 — Rewrite the 14 test assertions against `methods[0].params`

Do this **before** removing the synthesis, so you can see both shapes side by
side and map each assertion 1:1. The synthesis copies each param's
`kind`→`type`, `default`, and constraint (`minimum`, `maximum`, `enum`, …)
into `params_schema[p.name]`, and `default`/`None`-for-required into
`defaults[p.name]`. The canonical source is `methods[0].params[i]`, a list of
`MethodParamResponse`-shaped objects with fields `name`, `kind`, `default`,
`required`, `constraint` (with `min_value`, `max_value`, `enum_values`, …).

`tests/test_explainability_limitations.py`:

```python
# BEFORE (test_get_node_type_schema, line 475)
def test_get_node_type_schema(self) -> None:
    from sidecar.routes.node_types import get_node_type_schema
    schema = get_node_type_schema("cardre.random_forest_classifier")
    assert schema.node_type == "cardre.random_forest_classifier"
    assert "n_estimators" in schema.params_schema
    assert "feature_strategy" in schema.params_schema
    assert schema.defaults["n_estimators"] == 100

# AFTER
def test_get_node_type_schema(self) -> None:
    from sidecar.routes.node_types import get_node_type_schema
    schema = get_node_type_schema("cardre.random_forest_classifier")
    assert schema.node_type == "cardre.random_forest_classifier"
    params = {p.name: p for p in schema.methods[0].params}
    assert "n_estimators" in params
    assert "feature_strategy" in params
    assert params["n_estimators"].default == 100
```

## Failure mode

- **Codegen gate fails (`git diff --exit-code` non-empty):** you removed the
  model fields but did not run `python3 scripts/generate-openapi-types.py`.
  Run it and commit the regenerated files.
- **`params["objective"].constraint.enum_values` AttributeError:** the
  `MethodParamResponse` field names differ from the mapping table. Read the
  actual param model (grep `class MethodParamResponse` or the param type in
  `sidecar/models.py`) and use the real attribute names. The goal is the same
  coverage, not the same spelling.
- **A frontend test breaks on a missing mock field:** a mock still passes
  `params_schema`/`defaults`. Re-grep `frontend/src` for `params_schema` and
  `defaults` and remove every remaining stub.
- **`make preflight` reports generated-file drift:** the codegen produced
  different output than what's committed — re-run the script and stage the
  output.
