# Issue tracker: GitHub

Issues and PRDs for this repo live as GitHub issues.

## Conventions

- **Create an issue**: create a GitHub issue in `statcat2017/cardre`.
- **Read an issue**: read the issue body, comments, labels, author, and dates.
- **List issues**: list open GitHub issues, including labels and comments.
- **Comment on an issue**: post a GitHub issue comment.
- **Apply / remove labels**: edit the GitHub issue labels.
- **Close**: close the GitHub issue with a comment.

## Pull requests as a triage surface

**PRs as a request surface: no.**

## When a skill says "publish to the issue tracker"

Create a GitHub issue in `statcat2017/cardre`.

## When a skill says "fetch the relevant ticket"

Fetch the relevant GitHub issue from `statcat2017/cardre`.
